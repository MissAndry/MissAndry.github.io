Class vs. ID - What's the Difference?

The difference between classes and IDs can seem utterly arbitrary when dealing primarily with CSS. Either one references a specific style (written in a stylesheet) that can be added to HTML tags as attributes. However, there are pretty important differences!

Basically, a class is something you are going to use over and over again and an ID is something you are only going to use once. In other words, you give a tag a specific ID when you want to single it out and a class when you want to create faceless masses. 


<div id="sidebar">
	It's lkely that you'll only have one sidebar. If you have more than one, you 
	could name them something like "blogNavSidebar" and "generalNavSidebar" so 
	they remain unique, then call on a class for styling.
</div>
<p class="pretty">
	This will prettify the paragraph with font styles and backgrounds, but you 
	probably won't need to specifically find this paragraph. You'll also 
	probably want to use this style again.
</p>
<p>
	This paragraph will be pretty boring.
</p>
<p class="pretty">
	See, I knew you wanted pretty paragraphs again!
</p>

We can think of classes like an apartment layout and IDs like an apartment number. The same layout for apartments is likely to be used over and over again, but apartment numbers have to be unique. You wouldn't, for example, be able to find a single apartment by searching through layouts, but a number makes it a lot more simple. Other things that are unique to each apartment or apartment building would have an ID number without necessarily having a class.

This differentiation between class and ID becomes especially important in JavaScript. JavaScript only likes to look things up by apartment number (the ID), and it doesn't know what to do with non-unique identifiers. Fortunately, it'll ignore classes altogether - so you can reuse one class all you want!
