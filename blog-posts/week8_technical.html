Recursion

A method that uses recursion is a method that refers to itself. Any time you write a program with recursion, the program finds the first instance that has a return value and builds itself from that piece. Every piece of recursive code needs a condition with a return value in order for it to have that intial building block.

A very common example uses factorials. Since the factorial of a number n is the product of all numbers less than or equal to n, it provides a perfect example:

def factorial(num)
 if num != 0 && num <= 1
  1
 else
  num * factorial(num - 1)
 end
end

Within the if/else loop, the method factorial calls itself.

This might look completely bonkers at first. factorial isn't even finished being written when it's called, so how can it call upon itself?

If you notice, in the if statement there is a condition where num is less than or equal to 1. If that's the case, the outer method factorial returns 1. When num is greater than 1, factorial calls itself. Let's take a look at how that would be implemented. If we want to find the factorial of 3 with this method, we would call this:

factorial(3)  # => should return '6'

Once this gets to the if/else loop, it will multiply 3 by factorial(2), and within factorial(2), it will find factorial(1). Here's where the magic happens.

If we were write the long form of the the else statement above for factorial(3), it would end up looking something like this:

3 * factorial(2) * factorial(1)  # => factorial(2) will call factorial(1) in its else statement

The only one of these calls which immediately returns something is factorial(1). From there, the factorial method can build itself from the ground up. The else statement within factorial(2) unpacks to return:

2 * 1  # => Remember the code above! We want to retun num * factorial(num - 1)

Now that factorial(2) is returning something, factorial(3) should work no problem. The else statement becomes:

3 * 2

Which, of course, will return 6.
